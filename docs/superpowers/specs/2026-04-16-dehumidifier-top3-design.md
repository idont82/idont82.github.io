# 원룸·오피스텔 제습기 추천 묶음 설계

- 날짜: 2026-04-16
- 위치: `search/`
- 상태: 승인됨

## 목적

여름철 검색 유입을 노리는 제습기 콘텐츠 묶음을 만든다.
첫 기준 페이지는 `원룸·오피스텔 제습기 추천 TOP 3`로 잡고, 이후 같은 주제군 안에서 상황별 추천 페이지와 비교형 페이지를 연속 확장한다.

이 묶음은 단일 페이지 성과보다 `제습기` 키워드군에서 사이트 전체의 관련성과 내부링크 밀도를 높이는 것을 목표로 한다.

## 콘텐츠 묶음

1. `원룸·오피스텔 제습기 추천 TOP 3`
2. `장마철 제습기 추천 TOP 3`
3. `빨래건조용 제습기 추천 TOP 3`
4. `제습기 전기세 적은 제품 비교`
5. `제습기 vs 에어컨 제습 비교`

## 우선순위

1. `원룸·오피스텔 제습기 추천 TOP 3`
2. `장마철 제습기 추천 TOP 3`
3. `빨래건조용 제습기 추천 TOP 3`
4. `제습기 전기세 적은 제품 비교`
5. `제습기 vs 에어컨 제습 비교`

앞의 3개는 구매 직전 검색 의도가 강한 추천형 키워드이고, 뒤의 2개는 비교 탐색형 키워드로 내부링크 허브 역할을 맡는다.

## 기준 페이지 포지션

첫 페이지는 `원룸·오피스텔 전체`를 대상으로 하는 `구매 직전 비교형` 콘텐츠로 설계한다.
입문 설명보다 `지금 살 제품을 빠르게 3개로 좁혀주는 페이지`라는 성격이 우선이다.

## 제목 전략

기준 제목은 아래 방향으로 고정한다.

- `원룸·오피스텔 제습기 추천 TOP 3 (2026) | 소음·제습량·가성비 비교`

핵심 키워드:

- `원룸 제습기 추천`
- `오피스텔 제습기 추천`
- `제습기 추천 TOP 3`

보조 키워드:

- `소형 제습기`
- `자취방 제습기`
- `제습기 소음`
- `빨래건조 제습기`

## 도입부 원칙

도입부는 이론형 설명을 길게 넣지 않는다.
원룸·오피스텔 사용자가 실제로 비교하는 항목만 빠르게 제시한다.

- 소음
- 하루 제습량
- 물통 크기 또는 배수 편의
- 이동 편의
- 빨래건조 겸용성
- 가격

## 비교표 설계

비교표는 아래 열을 기본으로 한다.

- 가격대
- 하루 제습량
- 물통 용량
- 소음 체감
- 이동 편의
- 추천 대상

표는 사용자가 스크롤 없이 `누가 어떤 제품을 사야 하는지`를 바로 판단할 수 있게 요약형으로 쓴다.

## TOP 3 카드 설계

각 제품 카드는 아래 정보를 공통으로 갖는다.

- 한줄평
- 장점 3개
- 아쉬운 점 2개
- 핵심 스펙 요약
- 구매 링크

문체는 성능 과장이 아니라 `어떤 사용자에게 맞는지`를 중심으로 쓴다.

## 상황별 추천 섹션

기준 페이지에는 아래 소제목을 둔다.

- 조용한 방에서 쓸 사람
- 빨래건조까지 같이 할 사람
- 가성비를 우선 보는 사람
- 좁은 공간에서 자주 옮겨 쓸 사람

이 섹션은 비교표를 다시 문장으로 해석해 주는 역할을 한다.

## FAQ 설계

기준 FAQ는 아래 질문을 기본으로 한다.

- 원룸에 제습기 몇 L가 적당한가요?
- 제습기 소음은 어느 정도면 괜찮은가요?
- 원룸에서 제습기와 에어컨 제습은 뭐가 다른가요?
- 빨래건조용으로도 같이 쓸 수 있나요?

FAQ는 SEO 목적도 있지만, 구매 망설임을 줄이는 역할이 더 크다.

## 내부링크 설계

기준 페이지 하단에는 아래 4개를 반드시 연결한다.

- `장마철 제습기 추천 TOP 3`
- `빨래건조용 제습기 추천 TOP 3`

- `제습기 vs 에어컨 제습 비교`

반대로 뒤에 만드는 페이지들에서도 `원룸·오피스텔 제습기 추천 TOP 3`를 허브로 재링크한다.

## 확장 페이지 차이점

### 장마철 제습기 추천 TOP 3

- 연속 제습 안정성
- 물통 크기
- 배수 편의
- 장마철 사용 패턴

### 빨래건조용 제습기 추천 TOP 3

- 제습량
- 바람 순환
- 건조 체감
- 실내 건조 효율

### 제습기 전기세 적은 제품 비교

- 소비전력
- 운전 모드
- 장시간 사용 부담
- 유지비 관점 비교

### 제습기 vs 에어컨 제습 비교

- 작동 방식 차이
- 상황별 장단점
- 무엇을 먼저 사야 하는지
- 비용과 활용도 비교

## SEO 설계

- 모든 페이지에 `title`, `meta description`, `canonical`, `robots`, Open Graph, Twitter 메타를 넣는다.
- 추천형 페이지는 `Article + FAQPage` 구조화 데이터를 쓴다.
- 비교형 페이지는 `Article + FAQPage`를 기본으로 하고, 필요하면 `BreadcrumbList`를 추가한다.
- 제목에는 연도와 핵심 키워드를 같이 넣는다.
- 본문 초반 2~3문단 안에 `원룸`, `오피스텔`, `제습기 추천`, `소음`, `전기세`, `빨래건조`를 자연스럽게 포함한다.

## 템플릿 원칙

추천형 3개 페이지는 하나의 공통 템플릿을 재사용한다.
달라지는 것은 다음뿐이다.

- 제목
- 인트로 강조 포인트
- 비교표 기준
- 상황별 추천 문구
- FAQ 일부

이 원칙을 유지하면 제작 속도가 빨라지고 내부 품질도 일정해진다.

## 검증 기준

- 제목과 메타 설명이 대상 키워드와 일치한다.
- H1 1개, 비교표 1개, TOP 3 카드 3개, FAQ 3개 이상이 있다.
- 관련 제습기 페이지 내부링크가 하단에 노출된다.
- 모바일에서 비교표와 카드가 읽힌다.
- 추천형 3개 페이지가 같은 구조를 유지한다.
